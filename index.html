Don't Hurt Yourself

Adopting AngularJS in Existing Applications

Alexandra Atzl | @alexandraatzl

UI Developer at EffectiveUI | Rochester, NY

The Problem

You're using Angular for the first time, to build a large web application. Or maybe you're rewriting an existing application in Angular. You and your team haven't used AngularJS before, so you end up neglecting a lot of what Angular has to offer.

You end up wrapping relevant code in controllers and creating routes...but that's about it. You continue to write most of your code the way you normally would, without AngularJS.

Common Pitfalls

  • Relying on the presence of global variables, rather than handling dependencies using dependency injection.
  • Failing to take advantage of Angular's built-in services.
  • Putting business logic in controllers, when it should be abstracted into a service.
  • Repeating base objects, constant values, etc. in each component
  • Failing to modularize your code.

Take Advantage of Everything Available

If you're given an entire toolbox with which to build a bookshelf, why would you insist on doing everything using only the hammer?

So where do we start?

Break up files.

Every component should have its own folder, and every controller/service/directive should have its own Javascript file.

  • extra time taken is worth it in the long run
  • can more easily find the file you need, etc.
  • service is a singleton, so we know we're only ever dealing with that one instance
  • Angular will tell us if another servie of the same name is created, so there's no chance we'll inadvertently overwrite the object or its properties

Create Services

Eliminate global variables by wrapping them in services.

Most of these can be wrapped in an Angular service, and you can use dependency injection to make them available in the controllers where you need them.

Example: Global Object


angular.module('app')
	.controller('MyCtrl', function($scope) {
		// What is moment? Where is it defined? What if it's missing?
		$scope.startTime = moment();
		$scope.endTime = moment().add(1, 'h');

		// What about d3? How do I know this exists and is the object I want?
		$scope.chart = d3.select('div').append('svg')
			.attr('width', 600)
			.attr('height', 300)
			.append('g');
	});

Example: Wrapped in a Service


angular.module('app')
	.service('moment', function($window) {
		var myMoment = $window.moment;

		return {
			getMoment: function() {
				return $window.moment;
			}
		};
	})
	.service('d3Service', function($window) {
		var myD3 = $window.d3;

		return {
			getD3: function() {
				return myD3;
			}
		};
	});

It works for custom objects, too!

Use this method for any custom global objects you've created as well.

Example: Global Object

							
var zoo = {
	animals: null,
	getAnimals: function() {
		// get animals here and set this.animals
	}
}

angular.module('app')
	.controller('ZooCtrl', function($scope) {
		// hope that zoo exists somewhere
		$scope.myAnimals = zoo.getAnimals();
	});

Example: Wrapped in a Service

							
	angular.module('app')
		.service('zooService', function(){
			return {
				animals: null,
				getAnimals: function() {
					// get animals here and set this.animals
				}
			};
		})
		.controller('ZooCtrl', function($scope, zooService) {
			// now we can leverage dependency injection; our code is more
			// modular and testable
			$scope.myAnimals = zooService.getAnimals();
		});

Use all the tools available to you

Angular's built-in services make life easier. Use them!

Example: Grabbing data from a server

							
	angular.module('app')
		.service('addressBookService', function(){
			return {
				data: null,
				getAddresses: function() {
					$.ajax({
						url: '/path/to/data',
						dataType: 'json',
						success: function(data) {
							// handle success here
						},
						error: function(error) {
							// handle error here
						}
					});
				}
			};
		});

Example: Using Angular's $http service

							
	angular.module('app')
		.service('addressBookService', function($q){
			var deferred = $q.defer();

			return {
				data: null,
				getAddresses: function() {
					$http({
						method: 'GET',
						url: '/path/to/data'
					}).then(function(response) {
						deferred.resolve(response);
					}, function(error) {
						deferred.reject(error);
					});

					return deferred.promise;
				}
			};
		})

Keep business logic out of controllers

Abstract this logic into services instead.

Utilize Angular Values & Constants

If you commonly need to use the same values, don't add them to every controller individually!

Example: Repeated Code


	angular.module('todoListApp')
		.controller('workToDos', function($scope) {
			var BASE_TODO = {
				dueDate: null,
				text: '',
				id: 0,
				isCompleted: false
			};

			$scope.baseWorkTodo = angular.copy(BASE_TODO);
			$scope.baseWorkTodo.project = {
				id: 1,
				text: 'ngVegas talk'
			};
		})
		.controller('personalToDos', function($scope) {
			var BASE_TODO = {
				dueDate: '2015-05-08',
				text: 'Respond to emails.',
				id: 0,
				isCompleted: false
			};

			$scope.basePersonalTodo = angular.copy(BASE_TODO);
			$scope.basePersonalTodo.location = 'Las Vegas, NV';
		});

Example: Using Angular values

							
angular.module('todoListApp')
	.value('baseTodoObject', {
		dueDate: null,
		text: '',
		id: 0,
		isCompleted: false
	})
	.controller('workToDos', function($scope, baseTodoObject) {
		$scope.baseWorkTodo = angular.copy(baseTodoObject);
		$scope.baseWorkTodo.project = {
			id: 1,
			text: 'ngVegas talk'
		};
	})
	.controller('personalToDos', function($scope, baseTodoObject) {
		$scope.basePersonalTodo = angular.copy(baseTodoObject);
		$scope.basePersonalTodo.location = 'Las Vegas, NV';
	});

Take things a step further

Do you have multiple controllers that do almost the same thing? Try creating a base controller and extending it for each use case.

More testable! Cleaner!

Example: 


	angular.module('app')
		.controller('baseCtrl', function($scope) {
			$scope.recordType = '';
			$scope.recordService = null;

			$scope.getRecords = function() {
				$scope.recordService.getRecords().then(function(response) {
					// handle response
				});
			};

			$scope.addRecord = function() {
				$scope.recordService.addRecord(record).then(function(response) {
					// handle response
				});
			};

			$scope.deleteRecord = function() {
				$scope.recordService.deleteRecord(record).then(function(response) {
					// handle response
				})
			};
		});

Example:


	app.controller('userRecordCtrl', function($scope, $controller, userService) {
		// inherit $scope of baseCtrl
		$controller('baseCtrl', {
			$scope: $scope
		});

	  // now override $scope properties as necessary;
		$scope.recordType = 'User';
		$scope.recordService = userService;
	});
